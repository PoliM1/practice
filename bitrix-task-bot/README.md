# OAuth and Webhook
Отчёт
OAuth-приложение - это когда приложение входит через Google, GitHub или VK, но не просит твой пароль. Оно получает специальный ключ и работает от твоего имени.
Webhook - когда внешний сервис (например, банк или Telegram) сам автоматически присылает данные на мой сайт, как только что-то происходит (оплата прошла, пришло сообщение и т.д.). Это как мгновенное уведомление.
OAuth - это про вход и доступ, а Webhook - про автоматические уведомления.
Что я сделала:
1.Изучила, как работает REST API Bitrix24 для создания задач. 
2.Настроила Python и библиотеку requests для отправки запросов к Bitrix24. 
3.Создала скрипт, который автоматически создаёт задачи с названием, описанием и ответственным пользователем. 
4.Настроила работу скрипта на автоповтор, чтобы задачи создавались через определённый интервал времени.
5.Проверила, что задачи действительно создаются в Bitrix24 и выводятся в консоль ID новых задач.
Результат:
Я создала работающий скрипт-робот, который может автоматически создавать задачи в Bitrix24 и показывает в консоли информацию о каждой созданной задаче.
