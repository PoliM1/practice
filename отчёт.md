# ОТЧЁТ
Отчёт о работе
1)Написала бэкенд: создала сервер с FastAPI, настроила виртуальное окружение (venv), написала обработчики запросов (@app.get, @app.post), создала API endpoints для получения и отправки данных, подключила CORS для фронтенда и запустила сервер на http://127.0.0.1:8000/.
2)Сделала фронтенд: разработала HTML-страницы с формами для ввода данных, подключила JavaScript для отправки AJAX-запросов к бэкенду, настроила обработку ответов и обновление интерфейса без перезагрузки страницы.
3)Добавила гифку, показала запуск сервера, работу форм, отправку данных и ответы API, сохранила в формате .gif и загрузила в репозиторий.
4)Создала свой первый MD-документ "# СОЗДАНИЕ ФРЕЙМВОРКА И БЕКЕНДА".
5)Я настроила интеграцию своего проекта с Bitrix24 через REST API. Для этого я создала входящий webhook в Bitrix24, выдала ему права на работу с задачами и получила webhook URL, который используется для вызова методов API.
6)Далее я разобралась со структурой REST-запроса для создания задачи и использовала метод tasks.task.add, в котором передаются поля TITLE и RESPONSIBLE_ID; именно этот метод позволяет создавать задачи в Bitrix24 через внешний Python-проект. Я также поняла, что webhook должен храниться в секрете, потому что он дает доступ к действиям на портале в рамках выданных прав.
7)Со стороны проекта я подключила FastAPI, настроила роут для отправки POST-запроса на создание задачи и вынесла логику обращения к Bitrix24 в отдельный клиентский класс. Для статических файлов я использовала app.mount("/static", StaticFiles(directory="static"), name="static"), чтобы корректно подключались изображения и другие ресурсы интерфейса.
8)В результате я смогла сделать рабочую форму в своём приложении: пользователь вводит название задачи, после чего сервер отправляет запрос в Bitrix24, и задача успешно создаётся. Успешный ответ API возвращает result.task.id, по которому можно понять, что задача была создана корректно.
